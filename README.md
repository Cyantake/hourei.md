# 法令アシスタント（hourei-md）

**e-Gov の法令データを、AI が読み違えない形に整えます。**
法律・政令・省令をまるごと渡せるので、条文をまたいだ参照を辿りきれます。

国内の全法令を横断検索したい方には向きません。
**所管する一本の法律を、政令・省令・別表まで含めて隅々まで読み込む方**のためのツールです。

---

## こんな経験はありませんか

法令を AI に読ませると、こんな間違いをします。

- **別表の行がずれる** — 結合セル（縦や横につながったマス）のある表を、AI は対応関係ごと取り違えます
- **「〇〇条を準用する」で止まる** — 読み替えたあとの条文を教えてくれません
- **「政令で定める」の先を見ない** — 法・政令・省令が別々だと、委任関係が切れてしまいます

原因のひとつは、AI 検索でよく使われる「文書を細切れにして、必要な部分だけを探す」仕組みです。条文どうしの参照が、切れ目をまたげません。

このツールは、**法律・政令・省令を一本ずつ、まるごと AI に渡せる形に整えます。** 細切れにしないので、参照を辿りきれます。

---

## できること

```
  e-Gov から             このツールで              AI に
  法令XMLを              Markdown に               まるごと渡す
  ダウンロード    ─→     変換              ─→
                        （2回クリック）
```

---

## 特長

### 1. 別表が読める

法令の別表は、ほとんどが結合セルだらけです。これを普通に表として書き出すと、こうなります。

**変換前のイメージ（結合セルをそのまま表にした場合）**

```
| 海技試験の種別         | 乗船履歴                   |          |                    |                |
|                        | 船舶                       | 期間     | 資格               | 職務           |
| 六級海技士（航海）試験 | 総トン数五トン以上の船舶   | 二年以上 |                    | 船舶の運航     |
| 五級海技士（航海）試験 | 総トン数十トン以上の船舶   | 三年以上 |                    | 船舶の運航     |
|                        | 総トン数二十トン以上の船舶 | 一年以上 | 六級海技士（航海） | 船長又は航海士 |
```

最終行の「**どの試験の話なのか**」が空欄になっています。AI はここで前の行と結びつけられず、行がずれます。

**このツールの出力（実際の変換結果）**

```
列構成: [海技試験の種別] × [乗船履歴（船舶）] × [乗船履歴（期間）] × [乗船履歴（資格）] × [乗船履歴（職務）]
エントリ:
- 海技試験の種別=五級海技士（航海）試験, 乗船履歴（船舶）=総トン数十トン以上の船舶, 乗船履歴（期間）=三年以上, 乗船履歴（資格）=（なし）, 乗船履歴（職務）=船舶の運航
- 海技試験の種別=五級海技士（航海）試験, 乗船履歴（船舶）=総トン数二十トン以上の船舶, 乗船履歴（期間）=一年以上, 乗船履歴（資格）=六級海技士（航海）, 乗船履歴（職務）=船長又は航海士
```

**一行だけ切り出しても、意味が完結しています。** 空欄も `（なし）` と明示されるので、読み飛ばしが起きません。

縦の結合・横の結合はもちろん、**罫線だけで暗黙的に結合されているセル**も検出して解きほぐします。

### 2. 「読み替え」を整理して示す

準用規定の読み替えは、人間が読んでも骨が折れます。これを、対応表の形に開きます。

```
【読替え表】
規定: 第三条の三第一項第二号、第三号及び第四号、同条第二項第三号、第四号及び第五号並びに第三条の六第一号ニ
  「海技免許講習」 → 「海技免状更新講習」
規定: 第三条の三第一項第三号並びに同条第二項第三号及び第四号
  「法別表第一」 → 「法別表第二」
```

AI に渡す前に、**人間が読んでも分かる形**になります。

### 3. 変換できないものは、できないと書いて残す

様式や図は、テキストに変換できません。このツールは、それを黙って消しません。

```
> 〔附則様式：テキスト変換不可。原典を参照してください〕
```

**どこに何があったのかが残る**ので、「載っていない」と「変換できなかった」を取り違えずに済みます。

### 4. 手元だけで完結する

- **インストール不要** — Python の標準機能だけで動きます（追加のライブラリは一切不要）
- **登録不要・費用ゼロ** — 外部サービスへのアカウント登録はありません
- **変換はすべて手元の PC で行われます** — ネット接続もアカウントも不要です
- **スクリプトは全文が読めます** — 何をしているか、その場で確認できます

AI に渡すのは、**e-Gov で公開されている法令の条文だけ**です。手元の資料や、個別の案件の内容を含める必要はありません。

---

## こんなときに

- **「この要件は、法・政令・省令のどこに書いてあるのか」を確定させたいとき**
  条文をまたいだ委任関係を、AI が辿って示します。

- **「〇〇条の規定を準用する。この場合において〜と読み替えるものとする」を読み下したいとき**
  読み替えたあとの条文がどうなるのかを、整理して答えます。

- **別表の要件を、条件ごとに突き合わせたいとき**
  結合セルの多い別表でも、行がずれません。

## 使い方

### 準備：Python を入れる

[python.org](https://www.python.org/downloads/) からインストールしてください（**3.9 以降**）。
Windows の場合、インストール画面の「**Add Python to PATH**」に必ずチェックを入れてください。

追加でインストールするものはありません。

### 手順 1. e-Gov から法令をダウンロードする

1. [e-Gov 法令検索](https://laws.e-gov.go.jp/) で、調べたい法令を開きます
2. ページ内の **XML ダウンロード** を選びます
3. **ZIP ファイル（圧縮ファイル）で保存されるので、必ず解凍（展開）してください**
4. 中に入っている **`.xml` ファイル**を取り出します

法律・政令・省令の 3 本すべてについて、同じ操作を繰り返します。

> 法令の名前で検索すれば見つかります。法令番号などを調べる必要はありません。

### 手順 2. XML をスクリプトと同じフォルダに置く

このリポジトリをダウンロードして解凍し、その中に `.xml` を入れます。

```
hourei-md/
├── 1_egov_xml_to_json.py
├── 2_egov_json_to_md.py
├── 405AC0000000149.xml   ← 置いた XML
├── 329CO0000000200.xml
└── 326M50000800091.xml
```

### 手順 3. 2 つのスクリプトを、順番にダブルクリックする

**必ず番号の順に実行してください。**

1. **`1_egov_xml_to_json.py`** をダブルクリック → `.json` ができます
2. **`2_egov_json_to_md.py`** をダブルクリック → `.md` ができます

XML は、いったん JSON という中間の形式を経由して変換します。**順番にダブルクリックするだけです。**

> 逆の順番で実行すると「変換対象のファイルが見つかりません」と表示されて終わります。あわてず、`1_` から実行し直してください。
> 途中でできた `.json` は、`.md` ができたあとは削除してかまいません。

### 手順 4. ファイル名を変える（**重要**）

できあがった `.md` は、`405AC0000000149.md` のような英数字の名前になっています。

**AI が「法律・政令・省令のどれなのか」を判断できるよう、次のように名前を変えてください。** この手順を飛ばすと、次のプロンプトが正しく働きません。

| 元の法令 | 変更後のファイル名（例） |
|---|---|
| 法律 | `法_法律_船舶職員.md` |
| 政令（施行令） | `政令_施行令_船舶職員.md` |
| 省令（施行規則） | `省令_施行規則_船舶職員.md` |

末尾の `船舶職員` の部分は、対象の法令に合わせて自由に変えてください。

### 手順 5. AI に渡す

1. `prompts/法令アシスタント.md` の中身をコピーして、AI に**最初に**貼り付けます
2. 続けて、手順 4 で名前を変えた **3 つの `.md` ファイルを添付**します
3. 質問を始めます

> **一度に渡す量について**
> 3 本すべてを渡すと、法令によっては AI が一度に読める上限を超えることがあります。
> そのときは、**まず「法律＋政令」の 2 本だけを渡してください。** 多くの場合、これで足ります。
> 省令の細目まで必要になった段階で、3 本目を足すか、長文に対応したモデルに切り替えてください。

## 同梱プロンプトについて

`prompts/法令アシスタント.md` は、変換した Markdown を AI に読ませるための指示文です。次のことを AI に約束させています。

- **「政令で定める」「省令で定める」と書かれていたら、必ずその場で該当ファイルを開いて確認する**
- **「前条」「次項」「別表第〇」も、その場で確認する**
- **「準用する」「読み替えるものとする」があれば、読み替えたあとの内容を明示する**
- **3 つのファイルに書かれていないことは、推測で補わない**
- **書かれていない場合は、「3 ファイル内に記載が無いため不明」とはっきり答える**

最後の 2 つが要です。AI は、それらしい答えを作ってしまうことがあります。**「知らないことは知らないと言う」**を明示的に指示しています。

このプロンプトは、対象の法令に合わせて自由に書き換えてください。

## 動作確認

法律・政令・省令の 3 ファイル（**合計およそ 29 万字**）を一度に読み込ませ、条文間の参照を辿れることを確認しています。

| | |
|---|---|
| 確認済みのモデル | Claude Opus 4.7 以降 |
| 確認日 | 2026 年 7 月 |
| 検証に用いた法令 | 船舶職員及び小型船舶操縦者法（法・施行令・施行規則） |

**長い文章をまとめて読めるモデルが必要です。** 上記以外は未検証です。

読み込める量はモデルによって異なります。参考までに、検証に用いた法令の分量は次のとおりです。

| ファイル | 文字数 |
|---|---|
| 法律 | 約 7.2 万字 |
| 政令（施行令） | 約 3.2 万字 |
| 省令（施行規則） | 約 18.5 万字 |
| **合計** | **約 28.9 万字** |

省令が突出して大きいことが分かります。**「法律＋政令」だけなら約 10 万字**なので、より多くのモデルで扱えます。

## 既知の制約

- **施行日・公布日は出力されません。** 出力の冒頭に載るのは法令名と法令番号だけです。ダウンロードした XML が**いつ時点の内容か**は、e-Gov 側でご確認ください。
- **様式・図は変換されません。** 変換できなかった箇所は、その旨を明記して残しています（検証に用いた省令では 35 箇所）。原文をご確認ください。
- **表の種類は、内容から自動で判定しています。** 列の構成が特殊な表では、見出し行がデータとして扱われるなど、意図しない結果になることがあります。
- **罫線による暗黙の結合は、2 行分までの検出です。** 3 行以上にまたがる結合は、解きほぐせない場合があります。
- **目次は変換されません。** 本文の章立てをご参照ください。

## 必ずお読みください

- 本ツールの変換結果、および AI の回答には、**誤りが含まれる可能性があります**
- **回答に示された条文は、必ず e-Gov の原文でご確認ください**
- ダウンロードした XML が**いつ時点の内容か**を、e-Gov 側でご確認ください（施行日は出力に含まれません）
- **様式・図は変換されません。** 変換できなかった箇所は、その旨を明記して残しています
- 本ツールは、**法的助言を提供するものではありません**

## 上級者向け

e-Gov 法令 API（Version 2）から JSON を直接取得している場合は、手順 1〜3 を飛ばして `2_egov_json_to_md.py` だけを実行できます。この経路では、公布日・施行日などのメタ情報も出力されます。

各スクリプトはコマンドラインからも実行できます。

```bash
python 1_egov_xml_to_json.py input.xml -o output.json
python 1_egov_xml_to_json.py laws/*.xml -o ./json_output/
python 2_egov_json_to_md.py
```
